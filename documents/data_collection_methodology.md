# 핵의학 통계 데이터 수집 방법론 (Methodology)

이 문서는 핵의학 통계 기초 자료를 수집하고 분류하는 3대 체계와 그 데이터 간의 매핑 로직을 정의합니다. 본 문서는 데이터 파이프라인 설계의 도메인 지식 기반이 되며, 지속적으로 업데이트됩니다.

---

## 1. 데이터 수집 3대 체계

핵의학 통계는 공급(허가), 행위(급여), 시장(비급여)의 세 가지 관점에서 수집됩니다.

### ① 심평원 급여행위 EDI (요양급여 청구 데이터)
- **정의**: 건강보험심사평가원(HIRA)의 요양급여 비용 청구 시 발생하는 데이터.
- **특징**: 가장 정확한 '실행 건수' 통계. EDI 코드(예: FB011, FB710 등)를 기준으로 집계.
- **주요 데이터 필드**: EDI 코드, 검사명, 실시 건수, 연도/월, 의료기관 종별 등.

### ② 비급여행위 (비급여 통계)
- **정의**: 건강보험이 적용되지 않는 검사 및 치료 항목.
- **특징**: 공공 데이터포털이나 심평원 오픈 API에서 '비급여 진료비' 항목으로 수집하거나 학회 자료를 통해 보완 필요.
- **수집 항목**: 신의료기술 단계의 검사, 선택적 고가 검사 등.

### ③ 식약처 방사성의약품 품목허가 (마스터 데이터)
- **정의**: 식품의약품안전처(MFDS)로부터 승인받은 방사성의약품 및 관련 품목(제너레이터, 키트)의 공식 목록.
- **기술적 수집 출원**: `식품의약품안전처_의약품 제품 허가정보 OpenAPI`
- **핵심 필터링 소스**: 
  - **ATC 코드**: `V09` (진단용 방사성의약품), `V10` (치료용 방사성의약품)
  - **분류번호**: `431` (방사성의약품)

---

## 2. [개발자용] 방사성의약품 마스터 테이블 구현 상세

개발자는 아래 가이드에 따라 마스터 데이터를 수집하고 DB에 적재합니다.

### A. 데이터 수집 전략 (Sync Logic)
1. **대상 API**: `getDrugPrdtPrmsnInq` (의약품 제품 허가정보)
2. **필터링 파라미터**: 
   - `atc_code`: `V09` (1차 수집), `V10` (2차 수집)
   - 만약 ATC 코드로 누락되는 품목이 있을 경우 `class_no=431`로 보완 수집.
3. **증분 업데이트(Upsert)**: `ITEM_SEQ`(품목기준코드)를 고유 키로 사용하여, 기존 데이터는 업데이트하고 신규 데이터는 삽입합니다.

### B. 품목 카테고리 자동 분류 규칙
수집된 텍스트 데이터를 분석하여 시스템이 자동으로 카테고리를 분류합니다.

| 분류 (Category) | 판단 규칙 (Heuristics) |
| :--- | :--- |
| **Generator** | 제품명에 '제너레이터' 포함 또는 ATC 코드가 특정 발생기 코드인 경우 |
| **Cold Kit** | 제품명에 '키트' 포함 또는 성상/용법에 '표지하여 사용' 문구 포함 |
| **Finished** | FDG 등 가공 없이 즉시 투여 가능한 완제품 |

### C. 마스터 테이블 상세 설계 (mfds_radpharm_master)

| 필드명 | 타입 | 설명 | API 매핑 항목 |
| :--- | :--- | :--- | :--- |
| `item_seq` | `TEXT (PK)` | 품목기준코드 | `item_seq` |
| `item_name` | `TEXT` | 제품명 | `item_name` / `Prdct` |
| `entp_name` | `TEXT` | 업체명 | `entp_name` |
| `atc_code` | `TEXT` | ATC 코드 | `atc_code` (V09, V10 등) |
| `category` | `TEXT` | 항목 분류 | (로직 분류: Generator, Kit, Finished) |
| `edi_code` | `TEXT` | 보험코드 | `edi_code` |
| `main_item_ingr` | `TEXT` | 주성분 | `main_item_ingr` |
| `item_permit_date`| `DATE` | 허가일자 | `item_permit_date` |
| `cancel_date` | `DATE` | 취소/만료일자 | `cancel_date` |

> **분류(Category) 판정 로직**: `item_name` 내 키워드 스캐닝 (`제너레이터`->Generator, `키트`->Kit, 그 외 완제품->Finished)

## 3. 방사성의약품 품목허가 - EDI 매핑 로직

식약처 허가 기반의 의약품 공급 데이터와 심평원의 행위(검사) 데이터를 연결하기 위한 핵심 매핑 로직입니다.

### 테크네슘(Tc-99m) 기반 검사의 특수성
테크네슘 검사는 **[99Mo/99mTc 제너레이터]**에서 추출한 테크네슘과 특정 질환용 **[콜드 키트]**를 표지(Labeling)하여 수행됩니다. 식약처에서는 이들을 각각 별개 품목으로 허가하므로, 이를 검사 행위와 매핑해야 합니다.

| 식약처 허가 품목 (Input) | 행위 매핑 (Intermediate) | 심평원 EDI 코드 (Output) | 비고 |
| :--- | :--- | :--- | :--- |
| Tc-99m Generator + MDP Kit | 뼈 스캔 (Bone Scan) | **FB011, FB012** | 전신 또는 국소 |
| Tc-99m Generator + MAA Kit | 폐 관류 스캔 (Lung Perfusion) | **FB021** | |
| Tc-99m Generator + MIBI Kit | 심근 관류 스캔 / 부갑상선 | **FB051, FB032** | 용도별 분기 매핑 필요 |
| FDG (완제품) | PET/CT (전신/부분) | **FB710 ~ FB724** | 완제품은 1:1 매핑 용이 |

### 매핑 프로세스 (Concept)
1. **의약품 식별**: 수집된 MFDS 품목 코드에서 성분명 및 제형 확인.
2. **조합 규칙 적용**: (제너레이터 이용 여부) + (키트 종류)에 따른 검사 카테고리 분류.
3. **EDI 코드 할당**: 분류된 카테고리를 심평원 표준 행위 분류 코드와 매핑.
4. **가중치/계수 적용**: 특정 키트 하나가 여러 EDI 코드로 나뉘는 경우(예: MIBI), 진료 지침이나 통계적 가중치를 기반으로 배분.

---

## 3. 업데이트 정책

- **검사 코드 변경**: 심평원 수가 개정 시 즉시 반영 (매월 확인).
- **신규 의약품**: 식약처 신규 품목허가 승인 시 매핑 테이블에 추가.
- **매핑 고도화**: 학회(KSNM)의 자문 및 실제 처방 데이터를 기반으로 매핑 정확도 상향.

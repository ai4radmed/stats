# Test Spec: web/tests/pages/master-list.test.ts

## 1. 테스트 목적
`src/pages/master-list.astro` 페이지가 서버 사이드에서 데이터를 올바르게 렌더링하는지 검증합니다.

## 2. 테스트 항목 및 검증 포인트

### ① 데이터 렌더링 검증
- **상황**: Supabase에서 의약품 목록 및 API 로그를 성공적으로 가져왔을 때
- **검증**: 
    - 테이블 상단에 '최종 동기화', 'API 호출', '트래픽 사용량(%)' 등의 상태 정보가 표시되는가?
    - 테이블 내에 12개의 모든 필드(No., 생성일시, 수정일시, EDI코드 포함)가 화면에 표시되는가?
    - 첫 번째 컬럼이 'No.'이며 인덱스 번호가 정상적으로 출력되는가?
    - 'EDI 코드'가 테이블의 가장 마지막(12번째) 컬럼으로 위치하는가?
    - 카테고리에 따른 적절한 CSS 클래스(badge)가 적용되는가?

### ② 빈 데이터 처리 검증
- **상황**: 데이터베이스에 데이터가 없을 때
- **검증**: 
    - "데이터가 없습니다" 또는 유사한 안내 메시지가 화면에 노출되는가?

### ③ 레이아웃 정합성
- **상황**: 페이지 로드 시
- **검증**: 
    - `<h1>` 타이틀이 "방사성의약품 마스터 리스트"로 정확히 출력되는가?
    - 메인 페이지로 돌아가는 링크가 존재하는가?

## 3. 테스트 환경
- **도구**: `vitest`, `happy-dom` (Astro 컴포넌트 유닛 테스트 환경)
- **목킹(Mocking)**: 
    - `src/lib/supabase-server.ts`의 `supabaseAnon`을 목킹하여 API 호출 없이 테스트.
